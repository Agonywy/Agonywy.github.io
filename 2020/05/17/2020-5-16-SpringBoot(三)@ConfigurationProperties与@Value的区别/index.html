SpringBoot(三):@ConfigurationProperties与@Value的区别

发表于 2020-05-17 分类于 Spring boot 阅读次数：
本文字数： 2.6k 阅读时长 ≈ 2 分钟

一：@ConfigurationProperties(prefix = “”)

作用：默认从全局配置文件中获取值，注入属性的值，告诉Spring将本类中的属性和配置文件中的相关配置进行绑定，prefix=””表示配置文件中哪个Key下面的属性进行一一映射，当然：只有这个组件是容器的组件，才能使用容器中的功能

二：@Value注解

在SpringBoot中，除了使用@ConfigurationProperties注入属性的值，还可以使用Spring的底层注解@Value注入属性的值

那么Spring底层的@Value注解到底是从哪里来的呢？其实还是来自配置文件！

就是xml配置文件中的这个value=””完成的赋值，同样使用@Value来帮助我们以注解的方式给字段赋值。

三：@ConfigurationProperties获取值与@Value获取值的区别呢？

@ConfigurationProperties @Value
功能 批量注入配置文件中的属性 一个个指定
松散绑定(松散语法) 支持 不支持
SpEl（#{}） 不支持 支持
JSR303数据校验 支持 不支持
复杂类型封装 支持 不支持

  1. 松散绑定：属性名的匹配规则 lastName，last-name与last_name都代表同一个字段

  2. JSR303数据校验：我们可以对属性进行校验，使用@Validated表示这个javaBean需要校验

复杂类型封装：@Value不支持Map，Object，数组等复杂数据类型的封装.

如果我们只是在某个业务逻辑中需要获取一下配置文件中的某项值，我们就使用@Value；

如果我们专门编写了一个JavaBea来和配置文件进行映射，就需要使用@ConfigurationPaoperties注解来完成，一次性将配置文件中的属性都注入进来。
